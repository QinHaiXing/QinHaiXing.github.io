Spark纸质笔记

  • 第 1 章主要介绍 了 Spark 的发展历程、起源和基本的运行原理，为后面章节学习 Spark 组件打下 基础。
  • 第 2 章介绍了 Spark 的环境搭建，从软件下载、解压到环境变量、相应参 数的设置等方面讲解了如何在本地搭建 Spark 环境。
  • 第 3 章主要介绍了 Python 编程语言、PySpark 的启动和日志的设置、PySpark 开发包的安装以及如何使用 开发工具 PyCharm 编写 Spark 应用。
  • 第 4 章主要介绍了 Spark RDD 弹性分布式 数据集，内容包括 RDD 的特点、创建 RDD 的过程、RDD 的常用算子以及 RDD 的持久化。
  • 第 5 章主要介绍了 DataFrame 与 SparkSQL 的区别与联系，并对常用 操作进行了阐释。
  • 第 6 章介绍了 Spark Streaming 的特点、不同数据源的加载、 DStream 输出与转换操作以及与 DataFrame 和 SQL 的互操作。第 7 章主要介绍 了机器学习的相关概念、机器学习的一般流程、Spark 机器学习库 MLlib 和 ML。
  • 第 8 章主要介绍了基于 Spark 的第三方开源图计算工具 GraphFrames，内容包括 图的概念、图的操作和图算法，并用实际案例进行了应用操作。
  • 第 9 章和第 10 章分别介绍了综合实例出租车数据分析和图书推荐系统，通过两个综合实例让读 者从理论阶段上升到实际应用层次，帮助读者更深入理解 Spark 的使用方法。

第一章

Hadoop基础回顾

  • HDFS：分布式存储系统，适合存储少量的大文件，通过将文件分块，每一块复制多个副本并存储到不同的机器中，来实现数据可靠性。
  • MapReduce：处理大量数据，Map阶段负责将任务划分为子任务处理，Reduce阶段汇总结果（不适合实时）。
  • YARN：资源调度框架，负责资源调度和任务监控

Spark是什么

  • Spark是一种基于内存的、分布式的大数据处理框架

Spark的四大特点

  • 迅速

    • 对同一份数据的反复迭代运算，Spark是内存缓存，数据只加载一次，但是Hadoop需要反复加载

  • 通用

    • Spark框架包含了多个紧密集成的组件，面向不同应用需求

  • 易用

    • Spark提供了支持多种语言的API，如Scala、Java、Python、R语言等

      （Spark框架本身是用Scala语言编写的）

  • 支持多种资源管理器

    • Spark可以非常方便地与其他的开源产品进行融合
    • 使用Hadoop的YARN，-处理HDFS或Hbase的数据
    • Spark支持多种运行模式：本地local运行模式、分布式运行模式

Spark与Hadoop的区别

  • Hadoop是分布式数据设施，将巨大数据储存在由普通计算机组成的集群中
  • Spark则是一个专门的数据处理工具，它不会进行分布式数据的存储
  • Hadoop使用MapReduce进行数据处理
  • Spark处理的数据可存在Hadoop的HDFS或Hbase中

  • Hadoop缺点：磁盘IO频繁，耗费时间；对于复杂问题不方便设计作业
  • Spark:将中间结果存到内存中，速度远远快过磁盘读取，但占用内存资源大

Spark与Hadoop计算的根本差异：

  • Spark作业间是基于内存通信
  • Hadoop作业间是基于磁盘通信

Spark的用途

  • 保险行业：通过使用Spark的机器学习功能来处理和分析所有索赔，优化索赔报销流程。
  • 医疗保健：使用Spark Core，Streaming和SQL构建病人护理系统。
  • 零售业：使用Spark分析销售点数据和优惠券使用情况
  • 互联网：使用Spark的ML功能来识别虚假的配置文件，并增强他们向客户展示的产品匹配
  • 银行业：使用机器学习模型来预测某些金融产品的零售银行客户的资料。
  • 科学研究：通过时间，深度，地理分析地震事件来预测未来的事件。
  • 地理空间分析：按时间和地理分析Uber旅行，以预测未来的需求和定价
  • Twitter****情绪分析：分析大量的推文，以确定特定组织和产品的积极，消极或中立的情绪

Spark误区

  • Spark是基于内存计算，其他的不是（×）
    • 所有的技术都是基于内存计算，Spark**是将中间结果缓存在内存中
  • Spark计算比MapReduce快100倍（×）
    • 在反复迭代逻辑回归计算中快100倍，其他场景不一定
  • Spark的存在将代替Hadoop（×）
    • Hadoop是可扩展的框架，拥有分布式存储，Spark也可以利用Hadoop**的存储系统

Spakr生态系统

  • Spark Core：内核，相当于MapReduce，核心为RDD
  • Spark SQL：用来操作结构化数据的组件。通过Spark SQL，用户可以使用SQL或者Apache Hive版本的SQL方言（HQL）来查询数据。
  • Spark Streaming：支持实时数据的流式计算
  • MLlib：提供了很多机器学习的算法
  • GraphX：分布式图形处理架构，可用于图表计算

Spark架构

Driver Program：

驱动。运行Application的main()函数并且创建SparkContext，主要负责：

  1. 将用户程序转换为作业
  2. 在Executor间调度任务
  3. 跟踪Executor的执行情况
  4. 通过特定UI查询运行情况

Cluster Manager：

  • Spark的集群管理器，主要负责资源的分配与管理

Worker：

Spark的工作节点，一个Worker执行在一台服务器上，主要负责：

  1. 创建Executor（执行器）
  2. 将资源和任务分配给Executor
  3. 同步资源信息给Cluster Manager

Executor：

执行计算任务的一个进程，主要负责：

  1. 运行各任务（Task），并将结果返回给驱动器
  2. 让任务可以利用缓存数据加速计算

Spark作业运行流程：

有向无环图（DAG）：

名词汇总：

Application Spark的应用程序，包含一个Driver program和若干个Executor
SparkContext Spark应用程序的入口，负责调度各个运算资源，协调各个Worker Node 上的Executor
Driver Program 驱动。运行Application的main()函数并且创建SparkContext
Executor 执行者。是Application运行在worker node 上的一个进程，该进程负责运行Task,并且负责讲数据存在内存或者磁盘上。每个Application会申请各自的Executors来处理任务
Cluster Manager 在集群上获取资源的外部服务（例如：Standalone、Mesos、Yarn）
Worker Node 集群中任何可以运行Application代码的节点，运行一个或多个Executor进程
Task 任务。运行在Executor上的工作单元
Job 作业。SparkContext提交的具体Action操作，常和Action对应
Stage 阶段。每个Job会被拆分很多组任务（task）,每组任务被称为stage,也称为TaskSet
DAGScheduler 根据Job构建基于Stage的DAG，并提交Stage给TaskScheduler
TaskScheduler 将Taskset提交给Worker node集群运行并返回结果
Transformations/Action Spark API的两种类型：Transformation返回值还是一个RDD；Action返回值不是一个RDD，而是一个集合。所有的Transformation都是采用的懒策略，如果只是讲Transformation提交是不会执行计算的。计算只有Action被提交才会被触发

RDD简介

官方概念：

  • 弹性分布式数据集（RDD），是只读的、分区记录的集合，只能基于在稳定物理存储中的数据集和其他已有的RDD上执行确定性操作来创建

解释：

  1. 弹性：

    自动切换存储方式，优先存内存，内存不足再存磁盘

    RDD可以存储任意类型、可大可小的数据

  2. 分布式：

    有时数据量很大，需分布在不同机器中计算

  3. 数据集：

    RDD是一个抽象数据集，不存储真实数据，只存储一些描述信息

    初代RDD：只存储真实数据的分区信息等

    子代RDD：存储描述“初代RDD做了什么操作得到自己”

  4. 只读 ：

    RDD是不可变的，如果需要在一个RDD上进行转换操作，则会生成一个新的RDD

  5. 分区记录：

    RDD是分区的，RDD里面的抽象数据是分布在多台机器上的Executor里面的

  6. 什么时候创建：

    根据文件系统中的指定数据集创建（初代RDD）

    由其他RDD经过一定操作得到（子代RDD）

RDD执行任务流程：

  1. 从文件系统中选择想要计算的数据，创建第一个RDD，此RDD记录数据的位置、分区等信息
  2. 根据需要进行一系列转换，如排序、过滤、去重、map等，每进行一次转换，创建一个新RDD，每个RDD仅记录下以下信息：谁传过来的，我要执行什么操作，我要传给谁
  3. 所有转换完成后，发起执行命令，RDD开始创建任务，读取数据，依次执行所有计算，得出结果

RDD总结：

  • 相当于一个代理，用户需要进行什么操作，代理先全部记录下来，当用户需要结果时，代理再通知工人们执行任务

RDD算子

map：接受一个函数作为参数，对每个元素执行这个函数的转换

示例：

rdd = sc.parallelize([“b”, “a”, “c”])

sorted(rdd.map(lambda x: (x,1)).collect())

结果：

[(‘a’, 1), (‘b’, 1), (‘c’, 1)]

flatMap：先执行map，再扁平化（合并成一个列表）

示例：

rdd = sc.parallelize([2,3,4])

rdd2 = rdd.map(lambda x: range(1,x))

rdd2.collect()

map结果：[[1], [1, 2], [1, 2, 3]]

rdd1 = rdd.flatMap(lambda x: range(1,x))

rdd1.collect()

flatMap结果：[1, 1, 2, 1, 2, 3]

mapPartitions：先将列表进行分区，再把每个分区作为一个整体处理

示例：

rdd = sc.parallelize([1, 2, 3, 4], 2) *#把1234分成两个分区，一个是12，一个是34*

def f(iterator): yield sum(iterator) *#*定义一个函数，传入一个迭代类型，进行求和

rdd.mapPartitions(f).collect()

执行结果：

[3,7] 即两个分区的求和结果

mapPartitionsWithIndex：在mapPartitions基础上，传入的函数可接收两个参数，第一个参数为分区编号，第二个为对应分区的元素组成的迭代器

rdd = sc.parallelize([1, 2, 3, 4], 4) *#分成4*个分区

def f(splitIndex, iterator): yield splitIndex *#*函数直接返回分区编号

rdd.mapPartitionsWithIndex(f).collect()

结果：

[0, 1, 2, 3] 即各个分区的编号

filter：用于过滤。接受一个函数作为参数，对每个元素判断，结果为True的保留，否则去除

示例：

rdd = sc.parallelize([1, 2, 3, 4, 5])

rdd.filter(lambda x: x % 2 == 0).collect() *#*保留偶数

结果：

[2, 4]

distinct：去重

示例：

rdd = sc.parallelize([1, 1, 2, 3])

rdd.distinct().collect()

结果：

[2， 1， 3]

union：求两个rdd并集，不去重，接受另一个rdd作为参数

示例：

rdd1= sc.parallelize([1,2,3])

rdd2= sc.parallelize([4,5,6])
rdd1.union(rdd2).collect()

结果：

[1,2,3,4,5,6]

intersection：求两个rdd的交集，去重

示例：

rdd1 = sc.parallelize([1, 10, 2, 3, 4, 5])
rdd2 = sc.parallelize([1, 6, 2, 3, 7, 8])
rdd1.intersection(rdd2).collect()

结果：

[1, 2, 3]

subtract：返回rdd1有，rdd2没有的元素，且不去重

示例：

rdd1 = sc.parallelize([1, 2, 2, 3, 4, 5])
rdd2 = sc.parallelize([1, 3, 5])
rdd1.subtract(rdd2).collect()

结果：

[2 , 2, 4]

sortByKey：根据key进行排序

示例：

tmp = [(‘a’,1), (‘b’,2), (‘1’,3), (‘d’,4), (‘2’, 5)]
sc.parallelize(tmp). sortByKey()
结果：

[(‘1’, 3), (‘2’, 5), (‘a’, 1), (‘b’, 2), (‘d’, 4)]

默认升序，如果需要降序，加参数ascending=True

sortBy：根据指定key进行排序，key由一个函数获取

示例：

tmp = [(‘a’,1), (‘b’,2), (‘1’,3), (‘d’,4), (‘2’, 5)]
sc.parallelize(tmp).sortBy(lambda x: x[0]).collect() sc.parallelize(tmp).sortBy(lambda x: x[1]).collect()
结果：

[(‘1’, 3), (‘2’, 5), (‘a’, 1), (‘b’, 2), (‘d’, 4)] #相当于sortByKey

[(‘a’, 1), (‘b’, 2), (‘1’, 3), (‘d’, 4), (‘2’, 5)]

mapValues：对value进行map操作

示例：

rdd = sc.parallelize([(“a”, [1,2,3]), (“b”, [4]),(“a”, [5,6])])
def f(x): return len(x)
rdd.mapValues(f).collect()
结果：

[(‘a’, 3), (‘b’, 1) , (‘a’, 2)]

计算每个key对应的value长度

flatMapValues：先进行mapValues，再扁平化

示例：

rdd = sc.parallelize([(“a”,[“x”,”y”]),(“b”,[“p”, “r”])])
def f(x): return x
rdd.flatMapValues(f).collect()
结果：

[(‘a’, ‘x’), (‘a’, ‘y’), (‘b’, ‘p’), (‘b’, ‘r’)]

groupByKey：将key相同的合并，一般与mapValues配合使用

示例：

rdd = sc.parallelize([(“a”, 1), (“b”, 2), (“a”, 3)])
rdd.groupByKey().mapValues(len).collect()

结果：
[(‘a’, 2), (‘b’, 1)]
rdd.groupByKey().mapValues(list).collect()

结果：
[(‘a’, [1, 3]), (‘b’, [2])]

keys & values：取键，取值

示例：

m = sc.parallelize([(1, 2), (3, 4)])

m.keys().collect()

m.values().collect()

结果：
[1, 3]

[2, 4]

join & leftOuterJoin & rightOuterJoin：根据key对两个RDD进行内连接、左外连接、右外连接

示例：

x = sc.parallelize([(“a”, 1), (“b”, 4)])

y = sc.parallelize([(“a”, 2), (“a”, 3 ), (“c”, 5)])

x.join(y).collect()

结果：

[(‘a’, (1, 2)), (‘a’, (1, 3))]

collect：返回RDD中所有元素

sc.parallelize([1 , 2 , 3 , 4]).collect()

结果： [1 , 2 , 3 , 4]

count：返回元素个数

sc.parallelize([1 , 2 , 3 , 4]).count()

结果： 4

reduce：通过指定方法对RDD中的元素进行聚合

sc.parallelize([1, 2, 3, 4, 5]).reduce(lambda a,b:a+b) #求和

结果：
15

take：返回RDD中的前几个元素，若指定数量大于元素个数，则功能与collect相同

sc.parallelize([1, 2, 3, 4, 5]).take(3)

结果：
[1,2,3]

5、takeOrdered：排序后再take

sc.parallelize([4, 6, 8, 2, 9]).takeOrdered(3)

结果：
[2, 4, 6]

foreach：遍历所有元素，让所有元素都执行一个函数，没有返回值

def f(x): print(x)
sc.parallelize([1, 2, 3, 4, 5]).foreach(f)

结果： *#注意这里的打印是print(x)产生的*

1

2

3

4

5

collectAsMap ：以字典形式，返回PairRDD中的键值对，如果key重复，则后面的value覆盖前面的

rdd = sc.parallelize([(1, 2), (3, 4),(1,4)])

rdd.collectAsMap()

结果：

{1: 4, 3: 4}

countByKey ：以字典形式，返回PairRDD中key值出现的次数

rdd = sc.parallelize([(“a”, 1), (“b”, 1), (“a”, 1)])

rdd.countByKey()

结果：

[(‘a’, 2), (‘b’, 1)]
